[Prompt Flow](https://microsoft.github.io/promptflow/) uses its own tracing APIs. OTLP export is configured through OpenTelemetry settings.

## OTLP configuration

```bash wordWrap
export OTEL_EXPORTER_OTLP_ENDPOINT=https://www.comet.com/opik/api/v1/private/otel
export OTEL_EXPORTER_OTLP_HEADERS='Authorization=<your-api-key>,Comet-Workspace=<your-workspace>,projectName=<your-project-name>'
```

## Example (Prompt Flow tracing API)

```python
from promptflow.tracing import start_trace

# Enable Prompt Flow tracing for your app/session.
start_trace()

# Run prompt flow pipeline as usual.
```

## Notes

- Prompt Flow traces and OTEL export behavior are controlled by Prompt Flow tracing integration and OTEL configuration.
- Keep `projectName` in OTEL headers for explicit Opik project routing.

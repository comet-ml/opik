[AG2](https://ag2.ai/) is an open-source programming framework for building AI agents and facilitating cooperation among multiple agents to solve tasks.

AG2's primary advantage is its multi-agent conversation patterns and autonomous workflows, making it ideal for complex tasks that require collaboration between specialized agents with different roles and capabilities.

## Getting started

Install the Opik SDK and AG2:

```bash
pip install -U opik "ag2[openai]"
```

Configure Opik credentials:

<Tabs>
    <Tab value="Opik Cloud" title="Opik Cloud">
        ```bash
        opik configure
        ```

        Or set environment variables:
        ```bash
        export OPIK_API_KEY=<your-api-key>
        export OPIK_WORKSPACE=default
        ```
    </Tab>
    <Tab value="Self-hosted instance" title="Self-hosted instance">
        ```bash
        export OPIK_URL_OVERRIDE=http://localhost:5173/api
        ```
    </Tab>
</Tabs>

## Using Opik with AG2

The native integration wraps AG2's built-in OpenTelemetry instrumentation and bridges all spans into Opik traces automatically — no manual OTel setup required.

### Example 1: Two-agent chat

```python
from autogen import AssistantAgent, UserProxyAgent, LLMConfig
from opik.integrations.ag2 import OpikInstrumentor

# Initialize the instrumentor — LLM calls are auto-instrumented
instrumentor = OpikInstrumentor(project_name="ag2-demo")

# Define LLM config
llm_config = LLMConfig(api_type="openai", model="gpt-4o-mini")

# Create agents
with llm_config:
    researcher = AssistantAgent(
        name="Researcher",
        system_message="You are a research assistant. Find key facts.",
    )
    writer = AssistantAgent(
        name="Writer",
        system_message="You are a writer. Summarize research clearly.",
    )

# Instrument agents for tracing
instrumentor.instrument_agent(researcher)
instrumentor.instrument_agent(writer)

# User proxy to drive the conversation
user = UserProxyAgent(
    name="User",
    human_input_mode="NEVER",
    code_execution_config=False,
)
instrumentor.instrument_agent(user)

# Run the conversation
result = user.initiate_chat(
    researcher,
    message="What are the benefits of renewable energy?",
    max_turns=3,
)
print(result.summary)
```

### Example 2: Group chat with AutoPattern

```python
from autogen import AssistantAgent, UserProxyAgent, LLMConfig
from autogen.agentchat import initiate_group_chat
from autogen.agentchat.group.patterns.auto import AutoPattern
from opik.integrations.ag2 import OpikInstrumentor

instrumentor = OpikInstrumentor(project_name="ag2-group-chat")

llm_config = LLMConfig(api_type="openai", model="gpt-4o-mini")

with llm_config:
    analyst = AssistantAgent(
        name="Analyst",
        system_message="You analyze data and provide insights.",
    )
    summarizer = AssistantAgent(
        name="Summarizer",
        system_message="You summarize findings into concise reports.",
    )

instrumentor.instrument_agent(analyst)
instrumentor.instrument_agent(summarizer)

user = UserProxyAgent(name="User", human_input_mode="NEVER", code_execution_config=False)
instrumentor.instrument_agent(user)

# Create and instrument the group chat pattern
pattern = AutoPattern(
    initial_agent=analyst,
    agents=[analyst, summarizer],
    user_agent=user,
    group_manager_args={"llm_config": llm_config},
)
instrumentor.instrument_pattern(pattern)

result, context, last_agent = initiate_group_chat(
    pattern=pattern,
    messages="Analyze the impact of AI on healthcare.",
    max_rounds=5,
)
```

## What gets traced

The integration automatically captures:

| AG2 Activity | Opik Span Type |
|---|---|
| `initiate_chat` / `run_chat` | `general` (root trace) |
| Agent `generate_reply` | `general` |
| LLM calls (`OpenAIWrapper.create`) | `llm` |
| Tool execution | `tool` |
| Code execution | `tool` |
| Speaker selection (group chat) | `general` |
| Human input | `general` |

LLM spans include model name, token usage (prompt/completion), provider, and input/output messages.

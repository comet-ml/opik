Claude's official telemetry setup is environment-variable based.

## When this guide applies

Use this guide if you run Claude tooling that emits OpenTelemetry telemetry and you want to route that telemetry into an Opik-compatible OTLP pipeline.

## Opik OTLP endpoint modes

For full endpoint/header details, see [Opik OpenTelemetry overview](/tracing/opentelemetry/overview).

<Tabs>
    <Tab value="Opik Cloud" title="Opik Cloud">
        ```bash wordWrap
        export OTEL_EXPORTER_OTLP_ENDPOINT=https://www.comet.com/opik/api/v1/private/otel
        export OTEL_EXPORTER_OTLP_HEADERS='Authorization=<your-api-key>,Comet-Workspace=<your-workspace>,projectName=<your-project-name>'
        ```

        Required headers:
        - `Authorization`
        - `Comet-Workspace`

        Optional headers:
        - `projectName` (recommended for deterministic routing)
    </Tab>
    <Tab value="Enterprise deployment" title="Enterprise deployment">
        ```bash wordWrap
        export OTEL_EXPORTER_OTLP_ENDPOINT=https://<comet-deployment-url>/opik/api/v1/private/otel
        export OTEL_EXPORTER_OTLP_HEADERS='Authorization=<your-api-key>,Comet-Workspace=<your-workspace>,projectName=<your-project-name>'
        ```

        Required headers:
        - `Authorization`
        - `Comet-Workspace`

        Optional headers:
        - `projectName`
    </Tab>
    <Tab value="Self-hosted instance" title="Self-hosted instance">
        ```bash wordWrap
        export OTEL_EXPORTER_OTLP_ENDPOINT=http://localhost:5173/api/v1/private/otel
        export OTEL_EXPORTER_OTLP_HEADERS='projectName=<your-project-name>'
        ```

        Required headers:
        - none by default (depends on your auth setup)

        Optional headers:
        - `projectName` (recommended)
        - auth headers if your instance enforces auth
    </Tab>
</Tabs>

## Claude telemetry configuration (official pattern)

Intent:
Enable Claude telemetry exporters.

Applies when:
You run Claude tooling that supports telemetry environment variables.

Minimal valid input:
- enable telemetry
- choose exporters
- set OTLP protocol + endpoint

```bash wordWrap
export CLAUDE_CODE_ENABLE_TELEMETRY=1
export OTEL_METRICS_EXPORTER=otlp
export OTEL_LOGS_EXPORTER=otlp
export OTEL_EXPORTER_OTLP_PROTOCOL=grpc
export OTEL_EXPORTER_OTLP_ENDPOINT=http://localhost:4317
# Optional if your collector/export path requires auth
export OTEL_EXPORTER_OTLP_HEADERS="Authorization=Bearer <token>"
claude
```

## Signal compatibility note

Claude monitoring docs currently describe metrics and events/logs signals. Opik's tracing views are span-oriented. If you need trace-level analysis in Opik, verify your telemetry path emits spans compatible with your Opik ingestion setup.

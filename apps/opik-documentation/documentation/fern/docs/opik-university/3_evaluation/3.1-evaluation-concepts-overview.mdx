## Understanding LLM Evaluation with Opik

This video introduces the fundamentals of LLM [evaluation](/v1/evaluation/overview) and why it differs from traditional machine learning [metrics](/v1/evaluation/metrics/overview). Unlike conventional ML evaluation that relies on accuracy and F1 scores, LLM [evaluation](/v1/evaluation/overview) requires assessing text qualities like relevance, accuracy, and helpfulness. You'll learn about Opik's systematic three-component evaluation framework and see how it enables quantitative performance measurement across hundreds of test cases.

## Key Highlights

- **Beyond Traditional [Metrics](/v1/evaluation/metrics/overview)**: LLM [evaluation](/v1/evaluation/overview) requires new approaches since outputs are text that must be assessed for qualities like relevance, accuracy, and helpfulness
- **Three-Component Framework**: Opik's [evaluation](/v1/evaluation/overview) system consists of [datasets](/v1/evaluation/manage_datasets) (example inputs/outputs), [metrics](/v1/evaluation/metrics/overview) (automated scoring methods), and [experiments](/v1/evaluation/evaluate_your_llm) (evaluation runs)
- **Comprehensive [Dataset](/v1/evaluation/manage_datasets) Management**: Collections of example inputs and expected outputs that represent your specific use cases and requirements
- **Flexible [Metrics](/v1/evaluation/metrics/overview) System**: From simple [heuristics](/v1/evaluation/metrics/heuristic_metrics) to sophisticated [LLM-as-a-judge](/v1/evaluation/metrics/g_eval) approaches for automated output scoring
- **Systematic Experimentation**: Each [experiment](/v1/evaluation/evaluate_your_llm) represents a specific LLM application configuration tested against [datasets](/v1/evaluation/manage_datasets) with defined [metrics](/v1/evaluation/metrics/overview)
- **Model Comparison Power**: Compare different models (GPT-3.5 vs Claude 3.5 vs GPT-4 vs Gemini) systematically on the same [datasets](/v1/evaluation/manage_datasets)
- **[Prompt](/v1/prompt_engineering/prompt_management) Template Testing**: [Evaluate](/v1/evaluation/evaluate_prompt) various prompt templates against [datasets](/v1/evaluation/manage_datasets) with specific models for optimization
- **Quantitative Decision Making**: Replace subjective judgments based on few examples with quantitative measurement across hundreds of test cases
- **Production Confidence**: Structured [evaluation](/v1/evaluation/overview) approach provides confidence before deploying LLM applications to production

## End-to-End UI-Based LLM Experimentation

This comprehensive video demonstrates how to conduct complete LLM [experimentation](/v1/evaluation/evaluate_your_llm) using Opik's UI in a 15-minute video.
Through a practical example of building risk-aware LLM applications, you'll learn the complete workflow from [dataset](/v1/evaluation/manage_datasets) creation to [experiment](/v1/evaluation/evaluate_prompt) comparison.
The video walks through testing different [prompt](/v1/prompt_engineering/prompt_management) strategies to ensure LLM outputs include adequate cautionary statements, using LLM-as-a-judge [metrics](/v1/evaluation/metrics/overview) for systematic [evaluation](/v1/evaluation/overview) without requiring any coding experience.

## Key Highlights

- **Complete UI Workflow**: Demonstrates end-to-end [LLM evaluation](/v1/evaluation/evaluate_prompt) entirely through Opik's interface, making it accessible to non-technical users and data scientists alike
- **Practical Use Case**: Real-world example of ensuring LLM applications provide adequate risk warnings, preventing potentially dangerous command executions like Docker volume deletions
- **[Dataset](/v1/evaluation/manage_datasets) Creation & Management**: Upload CSV files or use AI to synthetically expand [datasets](/v1/evaluation/manage_datasets) with risky prompts for comprehensive testing scenarios
- **System [Prompt](/v1/prompt_engineering/prompt_management) Testing**: Compare different prompting strategies including "Risky Rick" (minimal warnings), "Safety Sid" (comprehensive cautions), and baseline ChatGPT responses
- **Interactive [Playground](/v1/prompt_engineering/playground)**: Test multiple [prompt](/v1/prompt_engineering/prompt_management) variations side-by-side with configurable model parameters and [dataset](/v1/evaluation/manage_datasets) integration using template variables
- **[LLM-as-a-Judge](/v1/evaluation/metrics/g_eval) [Evaluation](/v1/evaluation/overview)**: Create custom [evaluation rules](/v1/production/rules) that automatically assess output quality using Boolean scoring (0/1) for systematic risk assessment
- **Automated [Experiment](/v1/evaluation/evaluate_your_llm) Comparison**: Run parallel tests across entire [datasets](/v1/evaluation/manage_datasets) and compare aggregated [metrics](/v1/evaluation/metrics/overview) to identify the best-performing [prompt](/v1/prompt_engineering/prompt_management) strategy
- **Production Cost Considerations**: Configure sampling and filtering options for [online evaluation](/v1/production/rules) to manage costs when running LLM-as-a-judge on production data
- **Scalable [Evaluation](/v1/evaluation/overview) Approach**: Move beyond manual spot-checking to systematic assessment across 20+ test cases, enabling data-driven decisions for production deployment

[Pydantic AI](https://ai.pydantic.dev/) is a Python agent framework designed to
build production grade applications with Generative AI.

Pydantic AI's primary advantage is its integration of Pydantic's type-safe data
validation, ensuring structured and reliable responses in AI applications.

Opik integrates natively with Pydantic AI to log a trace for every `agent.run`,
with nested spans for each model call and tool call — no logfire required.

## Getting Started

### Installation

To use the Pydantic AI integration with Opik, you will need to have `opik` and
`pydantic-ai` installed:

```bash
pip install --upgrade opik pydantic-ai
```

### Configuring Opik

Configure the Opik Python SDK for your deployment type. See the [Python SDK Configuration guide](/tracing/advanced/sdk_configuration) for detailed instructions on:

- **CLI configuration**: `opik configure`
- **Code configuration**: `opik.configure()`
- **Self-hosted vs Cloud vs Enterprise** setup
- **Configuration files** and environment variables

### Configuring Pydantic AI

In order to use Pydantic AI, you will need to configure your LLM provider API keys. For this example, we'll use OpenAI. You can [find or create your API keys in these pages](https://platform.openai.com/settings/organization/api-keys):

You can set them as environment variables:

```bash
export OPENAI_API_KEY="YOUR_API_KEY"
```

Or set them programmatically:

```python
import os
import getpass

if "OPENAI_API_KEY" not in os.environ:
    os.environ["OPENAI_API_KEY"] = getpass.getpass("Enter your OpenAI API key: ")
```

## Logging Pydantic AI calls

To track your Pydantic AI agents, call [`track_pydantic_ai`](https://www.comet.com/docs/opik/python-sdk-reference/integrations/pydantic_ai/track_pydantic_ai.html). Called with no arguments it instruments every agent; pass a single agent to instrument only that one.

```python
from opik.integrations.pydantic_ai import track_pydantic_ai

track_pydantic_ai()
```

## Practical Example

Now that everything is configured, you can create and run Pydantic AI agents:

```python
import nest_asyncio
from pydantic_ai import Agent
from opik.integrations.pydantic_ai import track_pydantic_ai

# Enable async support in Jupyter notebooks
nest_asyncio.apply()

# Enable Opik tracing for all agents
track_pydantic_ai()

# Create a simple agent
agent = Agent(
    "openai:gpt-4o",
    system_prompt="Be concise, reply with one sentence.",
)

# Run the agent
result = agent.run_sync('Where does "hello world" come from?')
print(result.output)
```

## Cost Tracking

The `track_pydantic_ai` integration automatically tracks token usage and cost for all supported LLM models used by your agents. For models that report prompt caching (e.g. Anthropic and Bedrock), cache read and creation tokens are captured so cost is priced with the cache discount.

Cost information is automatically captured and displayed in the Opik UI, including:

- Token usage details
- Cost per request based on model pricing
- Total trace cost

## Grouping traces into conversational threads using `thread_id`

Threads in Opik are collections of traces that are grouped together using a unique `thread_id`.

Pass the `thread_id` through the native Pydantic AI `metadata` argument on `agent.run` (or `run_sync`) using the `opik.thread_id` key. Opik reads it and sets it on the trace:

```python
result = agent.run_sync(
    "What is machine learning?",
    metadata={"opik.thread_id": "conversation-123"},
)
```

More information on logging chat conversations can be found in the [Log conversations](/tracing/advanced/log_chat_conversations) section.

## Enriching traces with metadata and prompts

The same `metadata` argument accepts a few reserved `opik.*` keys that Opik maps onto the trace and root span:

- `opik.metadata` — a dictionary of custom metadata.
- `opik.provider` — override the provider recorded on the run.
- `opik.prompts` — a list of Opik prompts used for this run, to link the trace to your [prompt library](/prompt_engineering/managing_prompts_in_code).

```python
import opik

prompt = opik.Prompt(name="assistant", prompt="Be concise, reply with one sentence.")

result = agent.run_sync(
    'Where does "hello world" come from?',
    metadata={
        "opik.metadata": {"experiment": "baseline"},
        "opik.prompts": [prompt.__internal_api__to_info_dict__()],
    },
)
```

## Combining with `@track`

If you wrap your agent call in an `@track`-decorated function — for example to capture a clean entrypoint with its own input/output — the Pydantic AI spans automatically nest under the `@track` span, producing a single trace:

```python
import opik
from pydantic_ai import Agent
from opik.integrations.pydantic_ai import track_pydantic_ai

track_pydantic_ai()

agent = Agent("openai:gpt-4o")

@opik.track
def run(question: str) -> str:
    return agent.run_sync(question).output

run('Where does "hello world" come from?')
```

The result is one trace with the `run` entrypoint as the root and the agent run / model spans nested underneath.
